# YourGov Android — WebView MVP

Single-activity Android shell that wraps the YourGov web app in a strict, secure WebView.

## Prerequisites

| Tool | Version |
|------|---------|
| Android Studio | Hedgehog 2023.1.1 or later |
| Android SDK | API 35 (Android 15) |
| JDK | 17 (bundled with AS) |
| Kotlin | 2.0.0 (via Gradle plugin) |
| AGP | 8.5.0 |

---

## Build steps

### Option A — Android Studio (recommended)

1. Open Android Studio → **File → Open**
2. Select the `android-mygov/` folder (this directory)
3. Wait for Gradle sync to complete
4. **Run → Run 'app'** (or Shift+F10)

Android Studio downloads the Gradle wrapper automatically on first sync.

> **Note:** The `gradle/wrapper/gradle-wrapper.jar` binary is excluded from version control.  
> Android Studio regenerates it during sync. If using command line only, run  
> `gradle wrapper --gradle-version=8.7` from inside `android-mygov/` first.

### Option B — Command line (Linux/Mac)

```bash
cd android-mygov
chmod +x gradlew
./gradlew assembleDebug
```

### Option C — Command line (Windows)

```cmd
cd android-mygov
gradlew.bat assembleDebug
```

## Run on emulator / device

**Emulator:**
- AVD Manager → create device with API 35 image
- Run → Run 'app'

**Physical device:**
- Enable Developer Options → USB Debugging
- Connect via USB
- Run → Run 'app' → select device

## App behaviour

| Action | Result |
|--------|--------|
| Cold launch | Loads `https://yourgov.solvx.uk/start` |
| Back button (hardware) | WebView back, or exits app if at root |
| Forward button | WebView forward |
| Refresh button | Reloads current page |
| External button | Opens current URL in device browser |
| Share button | System share sheet with current URL |
| External link clicked | Opens in device browser, never in-app |
| Offline / error | Full-screen error with Retry + Open Browser |

## Security

- HTTPS only — cleartext blocked via `network_security_config.xml`
- URL allowlist: only `yourgov.solvx.uk` loads in-app
- All other domains → external browser intent
- File access: OFF
- Mixed content: NEVER\_ALLOW
- SafeBrowsing: enabled
- No dangerous permissions (only `INTERNET`)
- WebContents debugging: disabled in all builds

---

## Known limitations

- **No offline caching** — network errors show error state; no PWA cache
- **No push notifications** — would require FCM and a server-side token
- **Portrait only** — landscape locked in `AndroidManifest.xml`
- **No deep-link handling** — app does not register URL schemes
- **gradle-wrapper.jar excluded** — must be regenerated by Android Studio or `gradle wrapper`
- **No release signing** — see below

## Release signing TODO

1. Generate a keystore:
   ```bash
   keytool -genkey -v -keystore mygov-release.jks \
     -alias mygov -keyalg RSA -keysize 2048 -validity 10000
   ```
2. Add to `app/build.gradle.kts`:
   ```kotlin
   signingConfigs {
       create("release") {
           storeFile = file("../mygov-release.jks")
           storePassword = System.getenv("KEYSTORE_PASS")
           keyAlias = "mygov"
           keyPassword = System.getenv("KEY_PASS")
       }
   }
   ```
3. Reference in `buildTypes.release { signingConfig = signingConfigs.getByName("release") }`
4. Build: `./gradlew assembleRelease`
5. Output: `app/build/outputs/apk/release/app-release.apk`

**Never commit the `.jks` keystore to git.**
