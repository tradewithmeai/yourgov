# Gift: Country Adapter Starter Pack

Purpose: help an agent or user explore building a MyGov-style civic explainer for another country.

## First steps

1. Read `docs/COUNTRY_ADAPTER_ROADMAP.md`.
2. Read `docs/global-feasability-study-deep-research-report.md`.
3. Pick one country and classify it by capability tier:
   - Tier A: full map + vote/source lens possible
   - Tier B: source records + explainer possible, map incomplete
   - Tier C: source links and transparency guide only
4. Identify public data sources, license terms, member records, vote records, and boundaries.
5. Write a country-specific adapter plan before coding.

## Build rule

Adapt to available public data. Do not mark a country as failed just because it cannot match the UK feature set.

## Suggested first target

India is a useful first test because it matters to the hackathon audience and has strong parliamentary data signals, but it still needs source and boundary validation before claims are made.
